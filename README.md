# Obsidian Google Drive Encrypted Sync

Плагин для Obsidian, обеспечивающий надежную двустороннюю синхронизацию заметок через Google Drive с **end-to-end шифрованием (E2EE)** на стороне клиента.

---

## ✨ Основные возможности

* **Клиентское шифрование (AES-256-GCM)**: Содержимое заметок, медиафайлы, названия файлов и структура папок шифруются локально на вашем устройстве с помощью ключа, полученного из вашего пароля (через PBKDF2). Google видит только плоский список зашифрованных бинарных файлов.
* **Безопасная OAuth 2.0 авторизация**: Использует локальный loopback-сервер (`http://127.0.0.1`) для безопасного входа через ваш собственный Google-аккаунт в соответствии с требованиями безопасности Google.
* **Защита от массового удаления (Deletion Guard)**: Защищает от случайной очистки облака или локального хранилища. Если удаляется более 5 файлов и одновременно более 50% базы знаний, плагин приостановит работу и попросит подтверждения (можно отключить в настройках для запланированного удаления).
* **Разрешение конфликтов**: Несколько стратегий на выбор:
  * Побеждает более новый файл по времени изменения.
  * Создание копии конфликтующего файла (автоматическое переименование).
  * Всегда доверять локальной версии.
  * Всегда доверять удаленной версии.
* **Автоматическая синхронизация**: Фоновая синхронизация по таймеру (интервал настраивается) или автоматический запуск при открытии Obsidian.
* **Гибкие исключения**: Поддержка glob-паттернов исключений (например, для игнорирования `.obsidian/workspace.json`, папки корзины `.trash/**` и т.д.).

## 🛠️ Установка

### Вручную (Ручная установка)
1. Скачайте файлы последнего релиза (`main.js`, `manifest.json`).
2. В вашей папке заметок (Vault) перейдите по пути `.obsidian/plugins/` и создайте там папку `obsidian-gdrive-encrypted-sync`.
3. Поместите скачанные файлы в эту папку.
4. Откройте Obsidian → Настройки → Сторонние плагины и включите **Google Drive Encrypted Sync**.

## 📖 Настройка Google Cloud (OAuth 2.0 Credentials)

Поскольку плагин работает напрямую с вашим личным облаком без использования сторонних серверов-посредников, вам потребуется создать собственный ключ авторизации в Google Cloud Console:

1. Перейдите в [Google Cloud Console](https://console.cloud.google.com/).
2. Создайте новый проект (например, `Obsidian Sync`).
3. Перейдите в раздел **APIs & Services → Library** (Библиотека API), найдите **Google Drive API** и включите его.
4. Настройте экран согласия **OAuth consent screen**:
   * Выберите тип **External** (Внешний).
   * Укажите название приложения (например, `obsidian-gdrive-sync`) и контактную почту.
   * На шаге **Test users** обязательно добавьте ваш собственный e-mail адрес аккаунта Google Drive, который вы будете использовать для синхронизации.
5. Создайте ключи доступа (**APIs & Services → Credentials → Create Credentials → OAuth client ID**):
   * **ТИП ПРИЛОЖЕНИЯ**: Выберите строго **Desktop application** (Десктопное приложение).
   * Нажмите **Create** и скопируйте полученные **Client ID** и **Client Secret**.
6. Вставьте скопированные Client ID и Client Secret в соответствующие поля в настройках плагина в Obsidian.

## 🔐 Настройка шифрования
При первой настройке плагина на первом устройстве придумайте и установите надежный пароль шифрования (не менее 8 символов).

> [!CAUTION]
> Пароль шифрования никогда не отправляется в сеть и нигде не хранится вне вашего устройства. Если вы его забудете, расшифровать ваши файлы на Google Drive будет **невозможно**. Запишите его в надежном месте.

## 💻 Синхронизация на новом (втором) устройстве
Для настройки синхронизации на новом устройстве:
1. Установите плагин на новое устройство.
2. Введите те же **Client ID** и **Client Secret** в настройках.
3. Установите **тот же самый пароль шифрования**.
4. Нажмите «Подключить» и войдите в аккаунт.
5. Нажмите кнопку **«Полная пересинхронизация»**, чтобы плагин скачал и расшифровал все файлы из облака, воссоздав структуру папок локально.
